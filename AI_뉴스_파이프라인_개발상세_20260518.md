## 2. 전체 파이프라인 아키텍처

```
[STEP 1] 키워드 & RSS 피드 설정 (관리자)
    ↓
[STEP 2] RSS 수집 (자동)
    → Google News 토픽 피드 + 직접 RSS 피드 병행
    → Google News 리다이렉트 URL → 실제 기사 URL 자동 해제 (3.9-I~)
    → 기간 필터(14일) + 키워드 필터 → 기사 본문 크롤링
    → quality_score 자동 설정: 본문 300자 이상=2, 미만=1
    → DB: news_raw 테이블
    ↓
[STEP 2.5] 🧹 품질 필터 (관리자 트리거 — 선택적)
    → quality_score=2 기사를 Gemini 2.5 Flash로 배치 검증 (30건씩)
    → 주제 무관 기사 삭제, 관련 기사 quality_score=3으로 승격
    ↓
[STEP 2.6] Gemini 2.5 Flash 중요도 랭킹 (자동 — STEP3 실행 시 내부 동작)
    → 수집된 10건 → Flash가 중요도 1~5점 + 스팸 여부 판별
    → 3점 이상 + 스팸 아닌 기사만 → 상위 5건 선별
    ↓
[STEP 3] Gemini 2.5 Pro 인사이트 생성 (관리자 트리거)
    → 상위 5건 × 1,500자 본문 → 한국어 인사이트 1~2개 생성
    → 구조: 핵심 사실 → 구조적 의미 → 파급 효과
    → 503 시 자동 재시도(10s/20s) → 지속 실패 시 Flash 폴백
    → DB: news_insights 테이블 (status='draft')
    ↓
[STEP 4] 관리자 검수 & 발행
    → draft → review → published 상태 변경
    → 편집 모달 / 삭제 버튼 지원
    ↓
[STEP 5] 회원 화면 표시
    → /news-analytics: published 인사이트만 카드로 표시
```

## 3. 데이터베이스 스키마

### 3.1 `news_queries` — 카테고리별 수집 설정
```sql
CREATE TABLE news_queries (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    category    TEXT NOT NULL UNIQUE,  -- 'ai_infra', 'energy_infra', 'ai_service', 'physical_ai', 'ai_biology'
    query_text  TEXT,                  -- 키워드 (스페이스 구분)
    feed_url    TEXT,                  -- RSS 피드 URL (줄바꿈 구분, 여러 개 가능)
    is_active   BOOLEAN DEFAULT TRUE,
    created_at  DATETIME,
    updated_at  DATETIME,
    updated_by  TEXT
);
```

### 3.2 `news_raw` — 수집된 원본 기사
```sql
CREATE TABLE news_raw (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    category      TEXT NOT NULL,          -- 분야 카테고리
    title         TEXT NOT NULL,          -- 기사 제목
    url           TEXT UNIQUE,            -- 원본 URL (중복 방지, Google News 리다이렉트 해제된 실제 URL)
    source        TEXT,                   -- 출처 도메인 (예: techcrunch.com)
    published_at  DATETIME,              -- 기사 발행일 (RSS pubDate)
    collected_at  DATETIME DEFAULT NOW,  -- 수집 일시
    full_text     TEXT,                  -- 본문 크롤링 결과 (최대 5,000자)
    query_id      INTEGER REFERENCES news_queries(id),
    -- Phase 3.9-I 추가
    quality_score INTEGER DEFAULT 2      -- 1=본문없음, 2=본문있음(미검증), 3=Flash검증통과
);
```

**quality_score 규칙:**
| 값 | 의미 | 설정 시점 | UI 배지 |
|----|------|----------|--------|
| 1 | 본문없음 (차단/유료/오류) | 수집 시 자동 (full_text 300자 미만) | 본문없음 (회색) |
| 2 | 본문있음, 미검증 | 수집 시 자동 (full_text 300자 이상) | ✓본문 (초록) |
| 3 | Flash 검증 통과 | 품질 필터 실행 후 | ✦검증 (보라) |

### 3.3 `news_insights` — AI 생성 인사이트
```sql
CREATE TABLE news_insights (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    category          TEXT NOT NULL,
    insight_type      TEXT,                -- 'paradigm_shift', 'hidden_signal', 'value_chain', 'capital_flow', 'landscape_shift'
    title             TEXT NOT NULL,       -- 한국어 제목 (30자 이내)
    body              TEXT NOT NULL,       -- Markdown 형식 본문 (500~800자)
    status            TEXT DEFAULT 'draft', -- 'draft' | 'review' | 'published'
    related_news_ids  TEXT,               -- JSON 배열 [id1, id2]
    created_at        DATETIME,
    updated_at        DATETIME,
    created_by        TEXT,               -- 'gemini' 또는 관리자 email
    -- 미디어 메타데이터 (3.9-G에서 추가)
    image_url         TEXT,
    youtube_url       TEXT,
    related_tickers   TEXT,              -- JSON 배열 ["NVDA", "AMD"]
    is_synthesis      BOOLEAN DEFAULT FALSE,  -- 합성 인사이트 여부
    source_title      TEXT,
    source_url        TEXT,
    source_name       TEXT,
    source_date       TEXT
);
```

## 5. 핵심 함수 상세 설명

### 5.1 `_collect_rss_for_category(category, query_row, triggered_by, days=14)`

**역할**: RSS 피드에서 기사를 수집하고 `news_raw`에 저장

**처리 흐름**:
```
1. cutoff = 현재시각 - days일  (기간 필터 기준)
2. STOP_WORDS 제외 후 keywords 추출 (최대 8개)
3. query_row.feed_url에서 피드 URL 목록 파싱 (줄바꿈 구분)
4. 피드 URL마다:
   a. feedparser.parse(feed_url)
   b. 각 entry에 대해:
      - 날짜 필터: published_at < cutoff → skip
      - 키워드 필터: Google News 토픽 피드는 통과, 직접 피드는 제목+요약에서 1개 이상 매칭
      - [3.9-I] Google News 리다이렉트 해제: news.google.com URL이면 HEAD 요청으로 실제 URL 추출
      - URL 중복: news_raw에 이미 있으면 skip
      - 본문 크롤링: requests.get(url, timeout=6) → BeautifulSoup → <p> 추출 → 최대 5,000자
      - [3.9-I] quality_score 자동 설정: full_text 300자 이상 → 2, 미만/없음 → 1
      - news_raw에 INSERT
5. 반환: (added_count, searched_count, error_str, feed_details)
```

**RSS 피드 구성** (DEFAULT_FEED_URLS):
```
ai_infra:    Google News 토픽(AI/Tech) + TechCrunch AI + The Verge AI + Ars Technica + Reuters Tech
energy_infra: Google News 검색(AI energy power nuclear) + TechCrunch + Reuters Business + Ars Technica
ai_service:  Google News 토픽(AI/Tech) + TechCrunch AI + The Verge AI
physical_ai: Google News 검색(robotics autonomous AI) + TechCrunch Robotics + Ars Technica Cars
ai_biology:  Google News 검색(AI biotech drug genomics) + TechCrunch Biotech + STAT News
```

### 5.2 `_gemini_generate(model_name, prompt, timeout=90)`

**역할**: google-genai 패키지를 통한 Gemini API 일관 호출 헬퍼

> ⚠️ 2026-05-17 `google-generativeai` (deprecated) → `google-genai>=1.0.0` 교체 완료  
> ⚠️ 2026-05-19 503 UNAVAILABLE 자동 재시도 로직 추가 (3.9-I 버그픽스)

```python
_genai_client = genai_sdk.Client(api_key=GEMINI_API_KEY)

def _gemini_generate(model_name, prompt, timeout=90):
    """503 UNAVAILABLE 시 최대 3회 재시도 (10s, 20s 간격)."""
    last_err = None
    for attempt in range(3):
        try:
            response = _genai_client.models.generate_content(
                model=model_name, contents=prompt,
            )
            return response.text
        except Exception as e:
            last_err = e
            if ('503' in str(e) or 'UNAVAILABLE' in str(e)) and attempt < 2:
                time.sleep(10 * (2 ** attempt))  # 10s → 20s
            else:
                raise
    raise last_err
```

**generate/synthesis 라우트의 Pro→Flash 자동 폴백:**
```python
try:
    resp_text = _gemini_generate('gemini-2.5-pro', prompt)
except Exception as pro_err:
    if '503' in str(pro_err) or 'UNAVAILABLE' in str(pro_err):
        resp_text = _gemini_generate('gemini-2.5-flash', prompt)  # 폴백
    else:
        raise
```

### 5.3 `admin_news_pipeline_quality_filter()` — 품질 필터 (3.9-I 신규)

**역할**: quality_score=2인 기사를 Gemini Flash로 배치 검증하여 주제 무관 기사 삭제

```
1. target 카테고리(또는 전체) 지정
2. quality_score=2 기사 조회 (본문있음, 미검증)
3. 30건씩 배치로 Flash에 관련성 판별 요청:
   - 프롬프트: 카테고리 설명 + 기사 제목/본문 300자
   - Flash 응답: [{"id": N, "relevant": true/false}]
4. relevant=true → quality_score=3 승격
   relevant=false → DB에서 삭제
5. 반환: {total_kept, total_deleted, details:{cat: {kept, deleted}}}
```

### 5.4 `admin_news_pipeline_generate()` — 2-Tier Gemini 파이프라인

**Phase A: Gemini 2.5 Flash 랭킹**
```
입력: 최근 수집된 기사 10건 (제목 + 본문 500자)
프롬프트: 중요도 1~5점 + spam 여부 판별 요청
출력: JSON 배열 [{"id": N, "score": 1~5, "spam": true/false}]
필터: score >= 3 AND spam=false → 점수 내림차순 → 상위 5건
```

**Phase B: Gemini 2.5 Pro 인사이트 작성**
```
입력: 상위 5건 × 본문 1,500자
프롬프트 구조:
  - 역할: "Nerdy Oracle — Goldman Sachs 스타일 리서치 애널리스트"
  - 형식: ## 핵심 사실 → ## 구조적 의미 → ## 파급 효과
  - 분량: 500~800자 한국어 Markdown
  - 금지: "주가 예측", "매수 타이밍", "주가 임팩트", "단기 수익", "투자 추천"
출력: JSON 배열 [{"insight_type": "...", "title": "...", "body": "..."}]
저장: news_insights (status='draft')
```

## 10. 알려진 이슈 및 주의사항

### 10.1 RSS 수집 관련
- **Google News 토픽 피드 URL**: 하드코딩된 URL(`CAAqJggK...`)이 추후 변경될 수 있음 → 정기적으로 유효성 확인 필요
- **Google News 리다이렉트 URL**: ~~NotebookLM 파싱 불가~~ → **3.9-I에서 해결** — 수집 시 `requests.head(allow_redirects=True)`로 실제 기사 URL로 변환 저장
- **기사 본문 크롤링**: 일부 사이트(paywalled, JS 렌더링 필요)는 본문 수집 실패 → `full_text=None`, `quality_score=1`으로 저장
- **수집 시간**: 전체 5개 카테고리 수집 시 기사당 본문 크롤링으로 **5~10분** 소요

### 10.2 Gemini API 관련
- **비용**: Gemini 2.5 Pro는 Flash 대비 고비용 → Flash 사전 필터링으로 비용 절감 (10건 → 5건)
- **503 UNAVAILABLE**: ~~Pro 과부하 시 첫 번째 오류에 즉시 실패~~ → **3.9-I에서 해결** — `_gemini_generate()`에 최대 3회 재시도(10s/20s), 지속 실패 시 Flash 자동 폴백
- **JSON 파싱 실패**: Gemini가 JSON 대신 Markdown 코드블록으로 감싸서 응답하는 경우 자동 처리 코드 있음 (```json 제거)
- **타임아웃**: 복잡한 프롬프트는 90초 타임아웃 설정

### 10.3 DB 관련
- **`news_raw` 누적**: 중복 제거(URL 기준)를 하나, 계속 쌓임 → 주기적 정리 로직 필요 (현재 미구현)
- **로컬 vs 프로덕션**: 로컬은 SQLite (파일 기반), Railway는 PostgreSQL → `case()` 구문 등 일부 ORM 코드 dialect 차이 주의
